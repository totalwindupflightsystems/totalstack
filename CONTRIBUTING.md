# Contributing to TotalStack

TotalStack is a fully functional local AWS cloud stack — a LocalStack fork that emulates AWS services for development and testing. We welcome contributions that improve AWS service coverage, fix bugs, or enhance the developer experience.

## Setup

```bash
# Clone the repo
git clone https://github.com/totalwindupflightsystems/totalstack.git
cd totalstack

# Create Python venv and install dependencies
uv venv
uv pip install -e ".[runtime,test]"

# Verify installation
.venv/bin/python -m pytest specs/aws/.speclang/assembled/_tests/ -c /dev/null -x -q
```

Docker is required for running TotalStack as a container. Install Docker and then:

```bash
docker compose up -d
```

## Development Workflow

1. **Find a task** — Check the project board at `.coding-hermes/tasks.md` for open work items, or look at open GitHub issues.

2. **Write a failing test first** — Capture AWS behavior by running tests against real AWS:
   ```bash
   AWS_PROFILE=ls-sandbox TEST_TARGET=AWS_CLOUD SNAPSHOT_UPDATE=1 pytest tests/aws/services/<service>/test_<service>.py
   ```

3. **Implement the fix** — Find the provider in `totalstack/services/<service>/provider.py` and implement the handler.

4. **Verify against LocalStack** — Run the test without `TEST_TARGET=AWS_CLOUD` to validate against the emulator.

5. **Run full test suite** — Ensure no regressions:
   ```bash
   .venv/bin/python -m pytest specs/aws/.speclang/assembled/_tests/ -c /dev/null -x -q
   ```

6. **Code quality**:
   ```bash
   make lint
   make format
   ```

## Testing

### Integration Tests

Every service must have integration tests. Use `@markers.aws.validated` for tests verified against real AWS.

Key patterns (see `tests/aws/services/acm/test_acm.py` for reference):

- **Fixtures** — Use `aws_client`, `cleanups`, `snapshot`, `account_id`, `region_name`. Never hardcode account IDs or regions.
- **Snapshots** — Use `snapshot.match()` instead of plain `assert`. Add transformers for non-deterministic values before matching:
  ```python
  snapshot.add_transformer(snapshot.transform.key_value("FooArn"))
  snapshot.add_transformer(snapshot.transform.regex(dynamic_value, "<placeholder>"))
  ```
- **Resource cleanup** — Register resources with `cleanups` fixture for automatic teardown.
- **Running against AWS** — `AWS_PROFILE=ls-sandbox TEST_TARGET=AWS_CLOUD SNAPSHOT_UPDATE=1 pytest <path>`

### Shape Validator

The AWS shape validator (`development/aws-shape-validator.py`) verifies that every handler's input/output shapes match the AWS API specification. Run it with:

```bash
python3 development/aws-shape-validator.py --all
```

Current target: 76/76 services passing (52/76 as of July 2026).

## Implementing AWS Operations

### Provider Pattern

```python
from localstack.aws.api.<service> import <Service>Api
from localstack.services.<service>.models import <Service>Store, <resource>

class <Service>Provider(<Service>Api):
    @handler("CreateWhatever", expand=False)
    def create_whatever(self, context, request, ...):
        store = get_store(context)
        entity = WhateverRecord(id=short_uid(), ...)
        store.whatevers[entity.id] = entity
        return entity.to_dict()
```

### Reference: ACM Service

For a complete example, see the ACM service:
- **Provider:** `totalstack/services/acm/provider.py`
- **Tests:** `tests/aws/services/acm/test_acm.py`

### Key Conventions

- **Types:** Import from `localstack.aws.api` (auto-generated, don't modify)
- **Errors:** Use service-specific exceptions from the API module, or `CommonServiceException`
- **IDs:** Use `short_uid()` from `localstack.utils.strings`
- **ARNs:** Use helpers from `localstack.utils.aws.arns`
- **Logging:** `LOG = logging.getLogger(__name__)` at module top

## Hard Constraints

**NEVER:**
- Modify `*.snapshot.json` or `*.validation.json` files manually — they're generated by running tests against AWS
- Use plain `assert` in validated tests — always use `snapshot.match()`
- Create AWS resources directly in test bodies — use fixtures
- Hardcode account IDs or region names — use `account_id` and `region_name` fixtures
- Modify files in generated API specs (`specs/aws/`) — auto-generated from AWS API definitions
- Edit files in `localstack-core/` — upstream dependency, do not modify
- Add new project dependencies without discussion
- Run `git push` or modify repository history

## Pull Request Process

1. Ensure all tests pass locally (see Development Workflow above)
2. Run `make lint` and `make format`
3. Verify the shape validator shows no regressions for the service you modified
4. Open a PR against `main` with a descriptive title and summary
5. CI will run integration tests across Python 3.10, 3.11, and 3.12, plus the shape validator

## Code of Conduct

Be respectful. This is a collaborative project building an open-source AWS emulator. Focus on the work, keep discussions technical, and help each other grow.
